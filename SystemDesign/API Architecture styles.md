

**An API Architectural Style (or Pattern)** is a design paradigm. It dictates how your application logic is structured, how resources are exposed, and how the client and server agree to communicate _on top of_ those [[Network Protocols]]. It’s the delivery trucks and the packages they carry.

APU Architecture Style define
1. How clients talk to servers
2. How requests are structured
3. how responses are shaped
4. how contracts are enforced
