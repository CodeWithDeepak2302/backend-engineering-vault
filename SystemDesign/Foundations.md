
Client server Architecture
- Request response architecture.
- Server serves the clients.
- Client calls the [[DNS]] to get the IP of the server and then it actually calls the server
- Client then sends packets using [[Network Protocols]] to server. 


Polling vs Streaming

**Polling** 
client issues request to server again and again


**Streaming**
In case of instatnaeous requiredments like chatting. we need  chats instatnaneously.

Cient open a long lived connection with server and server keeps pushing the messages.
